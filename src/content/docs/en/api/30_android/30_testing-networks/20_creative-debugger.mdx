### Disabling the Creative Debugger

The following code shows you how to disable the creative debugger with the MAX <Abbr>SDK</Abbr>:

#### Java
```java
AppLovinSdk.getInstance( context ).getSettings().setCreativeDebuggerEnabled( false );
```

#### Kotlin
```kotlin
AppLovinSdk.getInstance( context ).settings.setCreativeDebuggerEnabled( false )
```

### Using the Creative Debugger Display <Abbr>API</Abbr>

In <Abbr>SDK</Abbr> versions 11.6.0 and above, you can display the Creative Debugger by making the following call:

#### Java
```java
AppLovinSdk.getInstance( context ).showCreativeDebugger();
```

#### Kotlin
```kotlin
AppLovinSdk.getInstance( context ).showCreativeDebugger()
```

## Creative <Abbr>ID</Abbr>

You can also retrieve the creative <Abbr>ID</Abbr> of displayed ads of various mediated networks.
You can do this either by using the creative debugger, or by using the MAX <Abbr>SDK</Abbr>.
The following code shows you how to do this with the MAX <Abbr>SDK</Abbr>:

#### Java
```java
String creativeId = ad.getCreativeId();
```

#### Kotlin
```kotlin
val creativeId = ad.creativeId
```

#### Java
```java
void onAdLoaded(final MaxAd ad)
{
  String networkName = ad.getNetworkName();
}

void onAdDisplayFailed(final MaxAd ad, final int errorCode)
{
  String networkName = ad.getNetworkName();
}
```

#### Kotlin
```kotlin
fun onAdLoaded(ad: MaxAd)
{
  val networkName = ad.networkName
}

fun onAdDisplayFailed(ad: MaxAd, errorCode: Int)
{
  val networkName = ad.networkName
}
```
