## How to Enable Test Ads in the AppLovin <Abbr>SDK</Abbr>

You can enable test ads for one or more <Glossary>GAIDs</Glossary> programmatically in the AppLovin <Abbr>SDK</Abbr>.

<Callout type="note">
With test ads, AppLovin will not track enabled impressions, clicks, and revenue.
You should only use such ads to test your app integration.
</Callout>

To enable test ads for a set of <Abbr>GAID</Abbr>s, pass a list of test device advertising <Abbr>ID</Abbr>s in AppLovin <Abbr>SDK</Abbr> settings.
Starting in <Abbr>SDK</Abbr> version 11.11.0, you must set test <Abbr>GAID</Abbr>s when you create the first <Abbr>SDK</Abbr> instance.
The following code snippets show how to do this in Java and Kotlin.

#### Java
```java
AppLovinSdkSettings settings = new AppLovinSdkSettings( context );
settings.setTestDeviceAdvertisingIds(Arrays.asList("«YOUR_GAID_HERE»"));
AppLovinSdk sdk = AppLovinSdk.getInstance( settings, context );
```

#### Kotlin
```kotlin
val settings = AppLovinSdkSettings(context)
settings.testDeviceAdvertisingIds = listOf("«YOUR_GAID_HERE»")
val sdk = AppLovinSdk.getInstance(settings, context)
```
